# 스키마 샘플 데이터 누락

## 문제
- "개발팀 평균 급여는?" → "조회 결과가 없습니다."
- 생성된 SQL: `WHERE departments.name = '개발팀'`
- 실제 DB 값: `'개발'` (팀 안 붙음)

## 원인
- 스키마에 테이블 구조만 있고 실제 데이터 값이 없음
- LLM이 부서명이 '개발'인지 '개발팀'인지 모름

스키마 출력 (당시):
```
TABLE departments:
  - dept_id (int)
  - name (varchar)
  - location (varchar)
```

## 해결 과정

### 시도 1: 프롬프트에 매핑 추가
- SQL Agent 프롬프트에 "개발팀은 개발, 영업팀은 영업..." 추가
- 결과: 검토만 (확장성 없음, 데이터 바뀌면 프롬프트도 수정해야 함)

### 시도 2: 스키마에 샘플 데이터 포함
- 파일: `core/database/connection.py:137-147`
- 변경 전:
```python
for col_name, data_type in columns:
    schema_text += f"  - {col_name} ({data_type})\n"

return schema_text.strip()
```
- 변경 후:
```python
for col_name, data_type in columns:
    schema_text += f"  - {col_name} ({data_type})\n"

# 샘플 데이터 추가
limit = "" if table_name in CODE_TABLES else f"LIMIT {SAMPLE_LIMIT}"
with self.engine.connect() as conn:
    samples = conn.execute(
        text(f"SELECT * FROM {table_name} {limit}")
    ).fetchall()

if samples:
    schema_text += f"  샘플 데이터:\n"
    for row in samples:
        schema_text += f"    {row}\n"
```
- 결과: 성공

스키마 출력 (수정 후):
```
TABLE departments:
  - dept_id (int)
  - name (varchar)
  - location (varchar)
  샘플 데이터:
    (1, '개발', '서울')
    (2, '영업', '부산')
    (3, '인사', '서울')
```

## 검증
- 테스트 입력: "개발팀 평균 급여는?"
- 기대 결과: SQL에 `name = '개발'` 사용
- 실제 결과: 정상 쿼리 및 답변 ✅
