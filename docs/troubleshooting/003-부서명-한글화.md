# 부서명/지역 한글화

## 문제
- "Engineering 팀 평균 급여는?" → 정상 작동
- "개발팀 평균 급여는?" → 조회 결과 없음
- 사용자는 한글로 질문하는데 DB는 영문

## 원인
- DB 데이터가 영문: `Engineering`, `Sales`, `HR`, `Seoul`, `Busan`
- 한글 질문과 매칭 안 됨

## 해결 과정

### 시도 1: DB 데이터 한글화
- 파일: `data/db_init/init.sql:60-61`
- 변경 전:
```sql
INSERT INTO departments (name, location) VALUES
('Engineering', 'Seoul'), ('Sales', 'Busan'), ('HR', 'Seoul');
```
- 변경 후:
```sql
INSERT INTO departments (name, location) VALUES
('개발', '서울'), ('영업', '부산'), ('인사', '서울');
```
- 결과: 성공

참고: "팀" 붙이지 않은 이유
- DB에는 `개발`, UI에서 필요하면 `+ '팀'`
- LLM이 "개발팀", "개발부서", "개발" 다 매칭 가능

## 검증
- DB 재시작 후 확인:
```sql
SELECT * FROM departments;
-- (1, '개발', '서울'), (2, '영업', '부산'), (3, '인사', '서울')
```
- 테스트 입력: "개발팀 평균 급여는?"
- 실제 결과: 정상 작동 ✅
