# 010-빈문자열-입력처리

## 증상
```
질문: ""  (빈 문자열)
응답: "개발팀 직원 수는 5명입니다."  (???)
```
빈 문자열 입력 시 Router가 임의로 해석하여 엉뚱한 답변 생성

## 원인
입력 검증 없이 바로 Router로 전달됨.
LLM이 빈 문자열을 "아무 질문"으로 해석하여 임의 응답.

## 해결
`core/agents/hr_agent.py`의 `query()` 메서드 시작 부분에 검증 추가:

```python
def query(self, question: str) -> AgentResult:
    # 빈 문자열 검증
    if not question or not question.strip():
        return AgentResult(
            success=False,
            answer="질문을 입력해주세요.",
            metadata={"agent_type": "VALIDATION"},
            error="Empty question",
        )

    # 이하 기존 로직...
```

## 검증
```
질문: ""
응답: "질문을 입력해주세요."  ✅

질문: "   "  (공백만)
응답: "질문을 입력해주세요."  ✅
```

## 업계 관행
- 빈 문자열/공백만 검증하는 것이 일반적
- "ㅋㅋㅋ" 같은 무의미한 입력은 LLM이 알아서 처리하도록 위임
- 과도한 검증은 오히려 정상 질문을 차단할 수 있음

## 교훈
- 입력 검증은 최소한으로 (빈 문자열 정도만)
- 의미 판단은 LLM에게 위임
- 검증 실패 시 명확한 에러 메시지 반환
