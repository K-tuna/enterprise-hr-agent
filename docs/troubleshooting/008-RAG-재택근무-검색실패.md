# 008-RAG-재택근무-검색실패

## 증상
```
질문: "재택근무 규정 알려줘"
응답: "해당 정보를 찾을 수 없습니다."
```

## 원인
FAISS 인덱스가 구 PDF(`01_회사규정.pdf`)로 생성되어 있었음.
새로 추가한 `회사규정.txt`에는 재택근무 내용이 있었지만, 인덱스에는 반영되지 않음.

## 진단 과정
1. `data/company_docs/회사규정.txt` 확인 → 재택근무 내용 존재
2. `data/faiss_index/` 생성 시점 확인 → 구 PDF 기준
3. RAG Agent가 FAISS 인덱스만 검색 → 신규 TXT 내용 미반영

## 해결
1. TXT를 PDF로 변환 (`02_회사규정.pdf`)
2. `experiments/exp_06_faiss_index.py` 수정:
   ```python
   # 파일명 변수화 (변경 시 여기만 수정)
   DOCS_FILENAME = "02_회사규정.pdf"
   ```
3. 인덱스 재생성:
   ```bash
   py -3.13 experiments/exp_06_faiss_index.py
   ```

## 검증
```
질문: "재택근무 규정 알려줘"
응답: "재택근무는 주 2회까지 가능하며..."  ✅
```

## 교훈
- 문서 변경 시 FAISS 인덱스 재생성 필수
- 인덱싱 대상 파일명을 변수로 관리하면 유지보수 용이
